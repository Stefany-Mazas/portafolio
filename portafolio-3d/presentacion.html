💻 Ejemplo de Código

ThreeSceneComponent -Inicialización Three.js

private initThree() {
  // Scene
  this.scene = new THREE.Scene();
  
  // Camera
  this.camera = new THREE.PerspectiveCamera(
    60, window.innerWidth / window.innerHeight, 0.1, 1000
  );
  this.camera.position.z = 8;
  
  // Renderer
  this.renderer = new THREE.WebGLRenderer({
    canvas, antialias: true, alpha: true
  });
  this.renderer.setSize(window.innerWidth, window.innerHeight);
  
  // Lighting
  const ambientLight = new THREE.AmbientLight(0xffffff, 0.5);
  this.scene.add(ambientLight);
  
  const pointLight = new THREE.PointLight(0x8b5cf6, 1, 100);
  pointLight.position.set(5, 5, 5);
  this.scene.add(pointLight);
}

🎯 Decisiones Técnicas

🤔

Three.js vs Babylon.js

Three.js tiene más documentación y ejemplos. Ecosistema mayor. Perfecto para demos.

��

GSAP vs native

GSAP ofrece animaciones más controladas y easing profesional. Vale la pena el peso extra.

🤔

Standalone vs NgModule

Angular 17+ recomienda standalone. Menos boilerplate, mejor tree-shaking.

🤔

Signals vs RxJS

Signals para estado local simple. RxJS para flujos complejos (HTTP). Usar ambos según necesidad.
